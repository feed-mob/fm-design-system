Getting Started

Work in Progress — This design system is currently under development and not ready for production use. 

# Install via npm
npm install @fm-agency/design-system

Then import in your JavaScript:

import '@fm-agency/design-system';
 
<!-- Add this to your HTML head -->
<script type="module" src="https://unpkg.com/@fm-agency/design-system/dist/fm-ds.js"></script>

Or download and use locally:

<script type="module" src="./dist/fm-ds.js"></script>
 
<!-- Just use the tags! -->
<fm-button variant="primary">Get Started</fm-button>

<fm-card hoverable>
  <span slot="header">Campaign Performance</span>
  Track real-time metrics across all active campaigns.
</fm-card>

<fm-badge variant="success" dot>Live</fm-badge>

That's it! FM Design System components work like native HTML elements.

/* Override theme tokens */
:root {
  --fm-color-primary: #0D9488;
  --fm-color-primary-light: #14B8A6;

  --fm-color-secondary: #1E293B;
  --fm-font-family: 'Inter', system-ui, sans-serif;
  --fm-radius-md: 8px;
}

The component is designed to work seamlessly with Ruby's Pagy gem. Simply pass the entire @pagy object and the component extracts page, pages, and series automatically. 

<!-- In your Rails controller -->
@pagy, @records = pagy(Product.all, items: 20)

<!-- In your ERB template -->
<!-- Option 1: Simplest - just pass the @pagy object -->
<fm-pagination pagy='<%= @pagy.to_json %>'></fm-pagination>

<!-- Option 2: With custom styling and SEO links -->
<fm-pagination
  pagy='<%= @pagy.to_json %>'
  variant="outline"
  href-template="/products?page=:page"
  show-first-last
></fm-pagination>

<!-- Option 3: With custom prev/next labels -->
<fm-pagination
  pagy='<%= @pagy.to_json %>'
  prev-label="Previous"
  next-label="Next"
  href-template="<%= url_for(page: ':page') %>"
></fm-pagination>

<!-- Option 4: Small size without first/last buttons -->
<fm-pagination
  pagy='<%= @pagy.to_json %>'
  size="sm"
  href-template="/items?page=:page"
></fm-pagination>

<!-- Option 5: Manual attributes (if you need granular control) -->
<fm-pagination
  page="<%= @pagy.page %>"
  pages="<%= @pagy.pages %>"
  series='<%= @pagy.series.to_json %>'
  variant="outline"
></fm-pagination>

Why this works: The pagy attribute accepts the entire Pagy object as JSON. The component automatically extracts:

  • pagy.page → current page
  • pagy.pages → total pages
  • pagy.series → page series array with :gap symbols

SEO & SSR: When href-template is provided, the component renders actual <a> tags for each page link. This enables:

  • SEO-friendly crawlable links
  • Right-click "Open in new tab" functionality
  • Graceful degradation without JavaScript

Attributes

fm-input

Basic inputs

Sizes

Variants

States

With Error

Attributes

NameTypeDefaultDescription
type string "text" Input type: text, password, email, number, search, tel, url
value string "" Current input value
placeholder string "" Placeholder text when empty
disabled boolean false Disables the input
readonly boolean false Makes input read-only
required boolean false HTML5 required attribute
error string "" Error message (shows error state when set)
size string "md" Input size: sm, md, lg
variant string "default" Visual style: default, filled, outlined

Events

NameDetailDescription
input { value: string } Fired on every keystroke
change { value: string } Fired on blur after value changes
focus — Fired when input receives focus
blur — Fired when input loses focus

Methods

NameReturnsDescription
focus() void Focus the input programmatically
blur() void Blur the input programmatically
clear() void Clear the input value

Parts

NameDescription
inputThe native input element
errorThe error message container
helperThe helper text container

Animations

  • Entrance: Fade in with subtle slide up (spring physics)
  • Focus: Border color transition + focus ring
  • Error: Shake animation when error appears

<!-- Variants -->
<fm-input variant="default"></fm-input>
<fm-input variant="filled"></fm-input>
<fm-input variant="outlined"></fm-input>

<!-- States -->
<fm-input value="Disabled" disabled></fm-input>

<!-- With error -->
<fm-input error="Must be at least $1,000"></fm-input>

<!-- Listen for events -->
<script>
  const input = document.querySelector('fm-input');
  
  input.addEventListener('input', (e) => {
    console.log('Current value:', e.detail.value);
  });
  
  input.addEventListener('change', (e) => {
    console.log('Final value:', e.detail.value);
  });
</script>

Attributes

NameTypeDefaultDescription
value string "" Current textarea value
placeholder string "" Placeholder text when empty
rows number 3 Minimum number of visible rows
disabled boolean false Disables the textarea
readonly boolean false Makes textarea read-only
required boolean false HTML5 required attribute
error string "" Error message (shows error state when set)
size string "md" Textarea size: sm, md, lg
variant string "default" Visual style: default, filled, outlined
autoresize boolean false Auto-expand height as user types
maxrows number 10 Maximum rows for auto-resize
maxlength number — Maximum character count
showcount boolean false Show character count (requires maxlength)

Events

NameDetailDescription
input { value: string } Fired on every keystroke
change { value: string } Fired on blur after value changes
focus — Fired when textarea receives focus
blur — Fired when textarea loses focus

Methods

NameReturnsDescription
focus() void Focus the textarea programmatically
blur() void Blur the textarea programmatically
clear() void Clear the textarea value
insertText(text) void Insert text at cursor position

Parts

NameDescription
textareaThe native textarea element
errorThe error message container
countThe character count element
helperThe helper text container

Animations

  • Entrance: Fade in with subtle slide up (spring physics)
  • Focus: Border color transition + focus ring
  • Error: Shake animation when error appears
  • Auto-resize: Smooth height transition
